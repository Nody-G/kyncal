# Kyncal — Context Memory

## Architecture
- **Stack**: Next.js 16 (App Router), TypeScript, Tailwind CSS v4, shadcn/ui (base-ui)
- **Stockage**: localStorage côté client (pas de backend)
- **Déploiement**: Vercel
- **Thème**: Dark par défaut, mode clair disponible
- **Exports**: jsPDF (PDF), xlsx (Excel), CSV natif

## Notes techniques
- shadcn/ui utilise `@base-ui/react` (pas radix) — pas de `asChild`, les Select ont `string | null` dans onValueChange
- Les contraintes d'enchaînement sont par défaut min=3, max=14 jours — à rendre configurables
- Le recalcul partiel est un placeholder (régénère tout pour l'instant)
- Jours de diffusion: chaque spectacle a `joursDiffusion: JourSemaine[]` — l'algorithme ne planifie que ces jours
- jsPDF: import dynamique `import("jspdf").then((m) => new m.default(...))`

## Données
- Cascadeur: nom, prénom, typeRepos (6j/1 ou 5j/1), roles[], absences[], actif
- Spectacle: nom, description, couleur, roles[], joursDiffusion[], actif
- RoleSpectacle: id, nom, nbCascadeursRequis
- RoleCascadeur: spectacleId, roleId, priorite (primaire/secondaire)
- Saison: nom, dateDebut, dateFin, spectacleIds[]
- Planning: saisonId, entrees[], dateGeneration
